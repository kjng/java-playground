Planning configuration data is used to configure various aspects of plans for this project. The following can be configured:

General: Specify how progress on work items is tracked, how estimate fields are rendered, and which iterations are used for product backlogs.

Plan Attributes: Specify which attributes are used for complexity and work item ranking.

Plan View Elements: Create and manage groupings, filters, and sort modes.

Plan Views: Allow you to manage plan views.

Plan Types: Allow you to manage plan types.

Top-Level Work Item Types: Specify which work item types should be used as top-level work item types in plans.

Work Environment: Specify work environment settings like days per week or hours per day.
